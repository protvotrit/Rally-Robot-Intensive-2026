# 🏎️ Проект: Автономный робот для Ралли

Добро пожаловать в репозиторий команды по разработке гоночного робота! Проект создается в рамках весеннего интенсива (30 марта – 3 апреля 2026 г.) для подготовки к соревнованиям «Ралли по коридору» (Регламент РобоФинист 6.1).

---

## 🛠 Технические требования (Check-list)
Чтобы пройти техническую комиссию, наш робот должен строго соответствовать следующим параметрам:

* **Автономность:** Полностью самостоятельное движение. Любое внешнее или телеуправление запрещено.
* **Конструкция:** Четырехколесный автомобиль с передним, задним или полным приводом. 
* **Рулевое управление:** Поворот осуществляется исключительно за счет управляемых передних колес вокруг вертикальной оси (схема Аккермана).
* **Габариты:** Робот должен быть меньше **500 мм** в длину, **400 мм** в ширину и **300 мм** в высоту.
* **Безопасность:** На передней части обязательно наличие мягкого бампера толщиной не менее **10 мм** (поролон, пенополиуретан, пористая резина).
* **Колеса:** Шины не должны удерживать лист бумаги А4 плотностью 80 г/м² дольше 2 секунд.

---

## 🧩 Электроника и компоненты
* **Контроллер:** Рекомендуется использование плат Arduino, ESP32 или Raspberry Pi Pico.
* **Датчики:**
    * Ультразвуковые или ИК-дальномеры для удержания дистанции от бортов (высота бортов — не менее 150 мм).
    * Датчики линии для распознавания черных меток перед препятствиями.
* **Привод:** Тяговый мотор на ведущую ось и сервопривод для управления углом поворота колес.

---

## 🚀 Алгоритм и препятствия
Для успешного прохождения трассы робот должен уметь:

1. **Остановка на «Стоп-линии»:** При обнаружении черной полосы шириной 250 мм робот обязан совершить полную остановку, не выходя проекцией за линию, и только затем продолжить движение.
2. **Преодоление «Камней»:** Участок с неровностями высотой до 35 мм.
3. **Проезд «Эстакады»:** Двускатная горка с углом подъема и спуска не более 20°.
4. **Реакция на знаки:** Перед «Камнями» и «Стоп-линией» нанесены черные линии (знаки оповещения) на расстоянии 1 метра, которые нужно использовать для смены алгоритма.

## 📁 Структура репозитория
* `/3D_Models` — файлы деталей (STL/M3D) для печати и моделирования.
* `/Code` — основные скетчи, библиотеки и алгоритмы управления.
* `/Docs` — схемы подключения, чертежи и полная версия регламента.
* `/Media` — фото и видео отчеты с заездов и тестов.
